// "Prop Drilling" -> É quando um componente no nível mais alto
// recebe 'props' e essas são passadas para outro componente de
// nível mais baixo e assim sucessivamente.

// Quando há uma cadeia de componentes muito grande, faz-se necessário
// usar um 'Context Manager'. Entretanto, esse projeto, por ser uma
// landing page, a prática de 'prop drilling' não será um problema,
// pois o projeto foi estruturado de modo a "gerenciar" essas 'props'
// de uma maneira "sustentável", ou seja, caso surjam algumas modificações
// nos componentes, não será complexo demais para fazer a manutenção
// dessas props.

// A prop 'children' do componente 'Base' é que será o 'html' dinâmico
// do site.
